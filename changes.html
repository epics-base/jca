Merging CAJ with JCA. Monitor payload size check at creation time. DBE_PROPERTY support. JNIException class made public. JNI CA Status check before DBR creation patch by Ike. Added support for -Djca.use_env=true - reads EPICS system env. variables. Mac OS X 10.6 (Intel) support. QueuedEventDispatcher and LastMonitorOnlyEventDispatcher concurrency improvements. PUT_ACKT/ACKS support added. Mac OS X (Intel) support. JNI processing fixed in case of error. QueuedEventDispatcher deadlock detection added (if queue is full and event is generated by event dispatcher queue). Clean implementation of JNI <code>AttachCurrentThread/DeattachCurrentThread</code>. <code>AttachCurrentThreadAsDaemon</code> used instead of <code>AttachCurrentThread</code> (to let JVM to exit if not using EPICS 3.14.9 or newer). JDK 1.4 or newer required by <code>AttachCurrentThreadAsDaemon</code>. Ported part of CAJ tests to test JCA. <code>JNIMonitor.removeMonitorListener()</code> implementation did actually add a monitor listener. Fixed. <code>JNIChannel.get()</code> threw a <code>NullPointerException</code> if channel not connected. Fixed to throw <code>CAException</code>. 64-bit platform support added (32-bit pointers replaced with 64-bit). JNI context initialization bug fixed. JNI global context message callback fallback fixed. Unused <code>getEnvPtr/freeEnvPtr</code> on non-BSD4.3 systems excluded from build. Priority support added.. <code>QueuedEventDispatcher</code> improved (blocks per PV). Added code to destroy event dispatcher to <code>JNIContext</code>. Added support for <code>ServerContext</code> and some changes to DBRs. Eclipse compatibility, exception callbacks, use of <code>epicsThreadPrivate</code>. (fix provided by Kenneth Evans, Jr.) Added <code>CAStatus.UNRESPTMO</code> (virtual circuit unresponsive) status. Added <code>ContextExceptionListener.contextVirtualCircuitException()</code> method. <code>gov.aps.jca.event.DirectEventDispatcher</code> - added exception handling of exceptions thrown from user code. Minor performance/memory improvement on enums types. Malformed JavaDocs fixed. I fixed a bug that was overwriting memory in <code>JNI.cpp</code>. (fix provided by Tom Pelaia) I added support for building and running JCA on Mac OS X. (update provided by Tom Pelaia) Simplified the OS scripts by using the EPICS environment variables to eliminate the hard coded paths so users will no longer have to edit these files for their setup. Now build system uses the <code>EPICS_BASE_RELEASE</code> and <code>EPICS_EXTENSIONS</code> environment variables that EPICS users use for referencing the EPICS base and extensions folders respectively. (update provided by Tom Pelaia) Added <code>gov.aps.jca.Context.getVersion()</code> method. Now users can retrieve context name and version information (at runtime). Added global context configuration defined in <code>JCALibrary.property</code>. Prefix <code>gov.aps.jca.Context</code> can be used instead of specific prefixes to set global context configuration. Of course, specific configuration overrides default configuration(s). Added <code>gov.aps.jca.JCALibrary.CHANNEL_ACCESS_JAVA</code> property. <code>gov.aps.jca.jni.JNI.ctxt_pool()</code> was always throwing a CAException due to the fact that the ca_poll() function in the C library always returns a timeout status code (fix provided by Eric Boucher). Major synchonization and performance fixes in <code>gov.aps.jca.event.QueuedEventDispatcher</code>. <code>gov.aps.jca.event.DirectEventDispatcher</code> minor performance improvement. <code>gov.aps.jca.JCALibrary.getInstance()</code> singleton pattern was missing synchronization. Removed unused imports in the code. Import of JCA 2.1.2 source code from APS pages.
